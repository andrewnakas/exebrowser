Run Age of Empires in your browser Hit or miss

Age of Empires is the test case people most want answered: can a real RTS classic run in a browser tab? The honest answer is sometimes, and not at full speed. The original 1997 Age of Empires and Age of Empires II are 32-bit Win32 games from the right era, so they're plausible — but they lean on DirectDraw and DirectSound and they're far more CPU-hungry than a card game, which is where browser-based emulation shows its limits.

AppsAge of Empires (1997), Age of Empires II: The Age of Kings (1999)
Best Wine variantTry Win32 (default) first, then 18R2 if it crashes on boot
Graphics2D DirectDraw — within reach; the 3D HD remasters are not
How to loadUpload the full install folder as a folder or zip — never just the EXE
Expected resultMay launch and play, but slowly; menus and small maps fare best

Set your expectations honestly

You're emulating an x86 CPU inside WebAssembly, which delivers roughly 10–40% of native speed. The original AoE games ran fine on a late-90s Pentium, so a fast modern laptop has headroom — but a busy late-game map with hundreds of units doing pathfinding will feel sluggish, and audio can stutter. Small skirmishes, the campaign tutorial, and menus are the most playable. This is a "revisit it / show someone" experience, not a competitive-multiplayer one.

Definitely the full install folder

AoE is not a single EXE — it needs its data, graphics, and sound files (the data, sound, campaign folders and various DLLs). Use the pick a folder or pick a zip option and upload the entire installed game directory so the executable finds everything beside it. Dropping just empires.exe or age2_x1.exe will fail immediately.

If it won't start

  • Crash on launch? Try the 18R2 variant — its different engine sometimes starts games the default chokes on.
  • Black screen? Open the Console and look for err: lines about a missing DLL, then add that file to your folder.
  • Wrong resolution / tiny window? Some builds accept a -1024 or windowed flag; the data-folder version is most forgiving.
  • The HD / Definitive editions won't work — they're modern 64-bit, Direct3D titles outside what this engine supports. Use an original 1997/1999 disc-era install.
