Classic DOS DOOM vs. source ports

This page runs the original DOOM.EXE DOS binary in DOSBox — the same executable that shipped on floppy disks in 1993. No source port, no Wine, just pure DOSBox emulation. WASD moves (remapped via DOSBox), Ctrl fires, Space opens doors. Arrow keys also work.

Want a source port?

DOOM's engine was open-sourced by id in 1997, spawning dozens of modernized ports like Chocolate Doom. These run the same shareware WAD with higher resolutions and fixed bugs. You can load a source port EXE alongside DOOM1.WAD using the "Load your own copy" option above.

Frequently asked questions

Can I play DOOM online for free with no download?

Yes. Episode one (Knee-Deep in the Dead) was released as free shareware in 1993 and is still freely redistributable. ExeBrowser runs the real Win32 build in your browser — nothing to install.

Is this real DOOM or a remake?

The real, original executable, running unmodified. ExeBrowser doesn't re-create the game; it runs id Software's actual Windows binary inside an x86 emulator and Wine, rendering to a browser canvas.

What are the controls for DOOM in the browser?

Click the game canvas to capture input. WASD moves (W=forward, S=back, A=turn left, D=turn right), Ctrl fires, Space opens doors, and the number keys switch weapons. Arrow keys also work. Press Esc to release the mouse back to the page.

Is it legal to play DOOM here?

Yes, for the shareware episode — id distributed it to be freely copied unmodified at no charge. The registered episodes and DOOM II are commercial and aren't included; buy those from id or Steam.

Why might DOOM run slowly in the browser?

An x86 CPU is emulated inside WebAssembly, so you get roughly 10–40% of native speed. DOOM is light enough to play at full speed on most modern laptops; older tablets and Chromebooks may feel slower.
