Play classic Winamp online in your browser Partial — UI yes, audio is iffy

Winamp — the player that really whipped the llama's behind — is one of the most nostalgic pieces of late-90s Windows software there is. The classic 2.x and early 5.x builds are small native Win32 programs, so the skinned interface, playlist editor, and equalizer all load under browser-based Wine. Audio playback is the catch: it routes through the emulated Web Audio path, and Winamp's output plugins don't always cooperate. Treat this as a fun nostalgia trip more than a reliable everyday music player.

Runs in your browser tab with WebAssembly + Wine — nothing is uploaded. Click the screen to capture input; press Esc to release the mouse.

We don't host Winamp here. It's freeware, not open-source, and its license doesn't grant redistribution rights. Download it from the official Winamp site or a reputable archive, and only run a copy you obtained legitimately.

How it works & what to expect

What works, honestly

The classic skinned Winamp window, the playlist editor, the equalizer, and skin loading all tend to render correctly — it genuinely looks like 2001 again. Where it gets shaky is actual sound output: Boxedwine routes audio to Web Audio, and Winamp's output plugins don't always agree with that path. You may get silence or stutter even when the UI is pixel-perfect. The visualizer and any plugins that probe hardware are also unreliable.

How to load it

Winamp ships as an installer, which won't run well here directly. Install it on a Windows PC (or extract a portable build), then upload the resulting Winamp program folder — winamp.exe plus its Plugins\ and Skins\ folders — using the folder picker. Drop a small MP3 in the same folder if you want to test playback. Stick with the Win32 (default) variant; it's the best fit for a small native player like this.

Why audio is the weak spot

Everything visual in Winamp is standard Win32 drawing, which Boxedwine handles well. Sound is different: Winamp uses output plugins (the classic out_wave / DirectSound path) that hand audio to the OS audio stack, and Boxedwine has to translate that to the browser's Web Audio API in real time inside an emulator running at roughly 10-40% native speed. That translation isn't lossless — you can hit silence, crackle, or buffer underruns. If playback fails, the UI is still worth a look for the nostalgia alone, and lightweight formats sometimes fare better than high-bitrate files.

Frequently asked questions

Does Winamp actually play music in the browser?

Sometimes. The interface, playlist, and equalizer load reliably, but audio output is hit-or-miss because Boxedwine has to route Winamp's output plugins through the browser's Web Audio path. You may get clean playback, stutter, or silence depending on the build and file. Treat it as a nostalgia demo rather than a dependable player.

Which Winamp version works best?

The classic 32-bit builds — Winamp 2.9x or an early 5.x "Lite/Full" — are the sweet spot. They're small native Win32 programs that render the iconic skinned window. Avoid modern 64-bit or DRM-laden builds; ExeBrowser is 32-bit Win32 only.

Do you host Winamp for download here?

No. Winamp is proprietary freeware and its license doesn't grant redistribution rights, so we link to the official site instead. Download it from winamp.com or a reputable archive and run a copy you obtained legitimately.

Can I use my old Winamp skins?

Yes — skin loading generally works. Upload the Winamp program folder with its Skins\ subfolder included via the folder picker, then switch skins from the UI as you normally would. The classic skinned look renders correctly even when audio doesn't.

Why isn't the visualizer working?

Visualization and any plugins that probe audio hardware or use the emulated sound path are unreliable here. The core skinned UI is rock-solid, but expect the visualizer, advanced output plugins, and hardware-touching add-ons to be flaky or silent.
