Play Microsoft Hearts online in your browser Single-player works well

The classic Microsoft Hearts — the trick-taking card game (mshearts.exe) that shipped with Windows from 3.11 all the way through Windows 7 — running in a browser tab. It is a tiny 32-bit GDI program with no DirectX, no 3D, and no .NET, which makes it one of the most reliable things to run in ExeBrowser. Single-player against the three computer opponents plays just like it always did. The one thing that won't work is the old "Microsoft Hearts Network" LAN play, but that's no loss — Microsoft already removed network play in the Windows XP version. Note that Hearts is proprietary Microsoft software, so we can't host it for you; bring your own copy from a Windows machine you own.

Runs in your browser tab with WebAssembly + Wine — nothing is uploaded. Click the screen to capture input; press Esc to release the mouse.

How it works & what to expect

ExeBrowser boots a real Wine 1.7.55 environment with a 32-bit x86 emulator compiled to WebAssembly, then runs the actual MSHEARTS.EXE binary — it is not a re-creation. Hearts is about the friendliest possible workload for this: a small, single-threaded GDI windowed app that just draws cards with the classic Windows card deck. There's no graphics acceleration to fall back from, so it renders cleanly and feels responsive even though the CPU underneath is emulated. Expect the real deal: enter your name, get dealt thirteen cards, pass three, and try to dodge the hearts and the Queen of Spades against three CPU players.

What works and what doesn't

  • Single-player vs. CPU — works. The full game against three computer opponents runs as designed: passing left/right/across/keep, shooting the moon, scoring to 100.
  • Mouse and keyboard — works. It's a point-and-click card game; click a card to play it. No input capture quirks like fullscreen shooters have.
  • Network / LAN play — does not work. The original "Microsoft Hearts Network" used NetDDE over a local network. The browser sandbox has no LAN peers, and in any case Microsoft dropped network play starting with Windows XP, so most builds you'll have are single-player only.
  • Sound — minimal. Hearts is a near-silent app, so there's nothing meaningful to miss here.

How to load it

  1. Locate MSHEARTS.EXE on a Windows PC you own — typically C:\WINDOWS\MSHEARTS.EXE on 95/98/Me/2000/XP, or C:\Windows\ on Vista/7.
  2. Copy it (and CARDS.DLL if your version has one) into a folder.
  3. Click ▶ Play above, then choose Load your own copy and select that folder.
  4. The player picks MSHEARTS.EXE as the entry automatically — click Run in Wine.
  5. Type a name when prompted and start passing cards. If it ever fails to launch, try the r18 Wine variant from the runtime menu.

Frequently asked questions

Can I play Microsoft Hearts online in my browser?

Yes — ExeBrowser runs the real mshearts.exe in your browser via Wine + WebAssembly, no install needed. Because Hearts is proprietary Microsoft software we can't host the file, so you load your own copy taken from a Windows PC you own.

Does Microsoft Hearts actually run well under emulated Wine?

Very well. It's a tiny 32-bit GDI app with no DirectX, 3D, or .NET, so it's one of the smoothest things ExeBrowser runs. Single-player against the three CPU players plays at full speed on a modern laptop.

Why can't I download Hearts directly from this page?

Hearts is part of Windows and is copyrighted by Microsoft — it isn't licensed for free redistribution, so hosting it would be piracy. You may legally copy MSHEARTS.EXE from a Windows installation you own and load it here.

Does the network/multiplayer mode work?

No. The classic "Microsoft Hearts Network" relied on NetDDE over a LAN, and the browser sandbox has no network peers. Microsoft also removed network play in Windows XP, so single-player vs. CPU is the experience here either way.

Which Windows version of Hearts should I use?

Any of them work — the Windows 95/98/Me/2000/XP builds are all small Win32 executables that run nicely. If one build refuses to launch on the default variant, copy a different version's MSHEARTS.EXE or switch to the r18 Wine variant.
