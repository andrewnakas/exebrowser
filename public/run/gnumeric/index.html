Run Gnumeric online in your browser Works well

Gnumeric is a free, open-source spreadsheet — the spreadsheet counterpart to AbiWord from the GNOME world. It is accurate, fast, lightweight, and runs reliably under browser-based Wine, which makes it a genuinely practical way to open or edit a spreadsheet on a machine where you cannot install Excel or LibreOffice. It opens .xls, .xlsx, .csv, .ods, and its own .gnumeric format, and writes most of them back out — so it doubles as a no-install file converter. Just hit play and start a sheet.

Runs in your browser tab with WebAssembly + Wine — nothing is uploaded. Click the screen to capture input; press Esc to release the mouse.

Gnumeric is GPL-2.0 open-source. Windows builds require an NSIS installer — install it on Windows, then load the installed gnumeric.exe and its DLLs using Load your own copy above.

Get a Windows build of Gnumeric

Gnumeric is free and open source (GPL-2.0). Download the official Windows build from gnumeric.org, then upload the program folder here to run it. We may host a ready-to-run package for one-click play.

How it works & what to expect

Why a spreadsheet is a good fit for browser Wine

Gnumeric is famous for the accuracy of its statistical functions and for being light on resources. It does not depend on modern GPU features, DirectX, or recent .NET, so older Windows builds run dependably in ExeBrowser. Recalculation is CPU work, so very large sheets will feel the emulation tax — but everyday spreadsheets are perfectly comfortable. Keep Win32 (default) selected; it is the right variant here.

Open, edit, and get your file back

Bundle the Gnumeric program folder together with the spreadsheet you want to open, upload as a folder, and edit inside the app. Save your work (Gnumeric can write CSV, XLS, and its own format), then click Download files written by this app to retrieve the saved file as a zip before you close the tab. This is a clean way to convert a stray .xls to CSV with no Office install at all.

  1. Click Play above and let Wine boot with Win32 (default).
  2. Use pick a folder to upload the Gnumeric program directory together with your spreadsheet file.
  3. Choose gnumeric.exe as the entry and run it.
  4. Edit your sheet, save it (CSV or XLS travel best), then click Download files written by this app before reloading.

When Gnumeric beats a full office suite

Modern office suites like LibreOffice are 64-bit and depend on features browser-based Wine cannot provide, so they will not boot here at all. Gnumeric's small footprint is exactly what makes it the practical choice in this environment. For the everyday jobs — open a spreadsheet someone emailed you, fix a few cells, re-export it; or pull a quick total out of a .csv on a Chromebook — it does the job without the weight. Reserve the heavy suites for a real desktop.

Frequently asked questions

Is Gnumeric free?

Yes. Gnumeric is open-source software released under the GPL-2.0 by the GNOME Project, free to download, use, and redistribute. You can get the official Windows build from gnumeric.org.

Can Gnumeric open Excel files in the browser?

Yes. Gnumeric reads .xls and .xlsx, along with .csv, .ods, and its own .gnumeric format. Upload the workbook alongside the program folder, edit it, then save and download the result.

How do I get my edited spreadsheet back out?

After you save inside Gnumeric, click Download files written by this app to pull the saved file out as a zip before you close or reload the tab. CSV and XLS are the most portable formats to save in.

Will large spreadsheets be slow?

They can be. Recalculation is CPU work and ExeBrowser runs Gnumeric through a WebAssembly x86 emulator at roughly 10-40% of native speed, so very large or formula-heavy sheets will feel the tax. Everyday spreadsheets run comfortably.

Why not just run LibreOffice or Excel instead?

Modern LibreOffice and Excel are 64-bit and rely on features browser-based Wine cannot provide, so they will not boot here. Gnumeric's light, 32-bit footprint is exactly what makes it the reliable spreadsheet choice in ExeBrowser.
