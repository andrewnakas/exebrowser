Play Rodent's Revenge online in your browser Works well on win3x — bring your own copy

Rodent's Revenge is the addictive little puzzle game from Microsoft Entertainment Pack 2 (1991): you're a mouse pushing blocks to corner the cats, who turn into cheese when they're trapped. It's tiny, turn-based, all 2D tile graphics — no DirectX, no 3D, no networking — which makes it a near-perfect fit for running on Wine + WebAssembly. The one catch: it's a 16-bit Windows 3.x program, so it needs ExeBrowser's win3x runtime, and because it's a Microsoft commercial title we don't host the game files — you load your own legally-owned copy.

Runs in your browser tab with WebAssembly + Wine — nothing is uploaded. Click the screen to capture input; press Esc to release the mouse.

How it works & what to expect

Rodent's Revenge is about as gentle a workload as Wine gets. There's no hardware rendering, no sound mixing to speak of, and the whole game is a grid of sprites that update a few cells at a time. On ExeBrowser's emulated x86 CPU you'll still get roughly 10–40% of native speed, but for a turn-based block-pusher that's effectively instant — the bottleneck is you deciding where to shove the next block, not the emulator.

Expect the original tiny window (it was designed for 640×480 Windows 3.1 desktops), crisp pixel graphics, and the familiar squeak-and-trap loop across its 50 levels. It will look exactly like 1991, because it is 1991 — this runs the real executable, not a remake.

Why you need the win3x runtime

This is the important bit. Rodent's Revenge is a 16-bit Windows 3.x application. Modern 64-bit Windows can't run it at all, and even ExeBrowser's everyday default (Win32) runtime won't load a 16-bit binary. ExeBrowser provides a dedicated win3x runtime for exactly these 1990–1994 programs, and that's what this page is set to use.

  • If the game refuses to start on the default engine, that's expected — make sure you're on the win3x variant (this page selects it for you).
  • The original package often needs a couple of helper DLLs alongside the EXE — commonly VBRUN100.DLL and WEPUTIL.DLL. If your copy includes them, keep the whole folder together when you load it.
  • Load the folder, not just the EXE, so Wine can find those DLLs and any data files next to it.

Loading your own copy, step by step

  1. Get a copy you legally own — an original Microsoft Entertainment Pack 2 disk you purchased, copied to your computer.
  2. Unzip it so the executable (often RODENT.EXE) sits in a folder with its helper DLLs.
  3. Click ▶ Play Rodent's Revenge above to boot the win3x runtime right on this page.
  4. Use Load your own copy → a folder and pick the unzipped Rodent's Revenge directory.
  5. Click Run in Wine, then click the game window to give it keyboard focus.

Controls are simple: the ↑↓←→ arrow keys move the mouse one square at a time, pushing any movable blocks in front of you. Box every cat in on all four sides to turn it into cheese, then eat the cheese for points.

Why won't Rodent's Revenge start on the normal runtime?

Because it's a 16-bit Windows 3.x program, not a 32-bit Win32 app. The everyday default runtime only loads Win32 binaries. This page uses ExeBrowser's win3x runtime, which is built specifically for early-1990s 16-bit Windows software like the Microsoft Entertainment Pack.

Is it legal to play Rodent's Revenge here?

Playing a copy you legally own is fine. Rodent's Revenge is still under Microsoft's copyright — it was sold in Microsoft Entertainment Pack 2 and was never clearly released for free redistribution, despite claims you may see elsewhere. Is this the real game or a remake?

Will it run smoothly in the browser?

Yes. It's a 2D, turn-based puzzle game with no DirectX, 3D, or networking, so even at emulated speeds it feels instant. The win3x runtime handles it comfortably on a normal laptop; very old tablets or Chromebooks may take a little longer to load the engine.
