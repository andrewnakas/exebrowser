Can Freeciv run in your browser? Won't run — Boxedwine lacks setjmp support required by this EXE

Freeciv is a free, open-source empire-building strategy game inspired by Civilization. Unfortunately the Win32 SDL 2.6.7 build can't run in ExeBrowser: the EXE uses setjmp/longjmp for exception handling, and neither Boxedwine build we host (default or 18R2) was compiled with setjmp emulation support. Boxedwine detects this at load time and aborts with "setjmp is required for this app but it was compiled into boxedwine."

Runs in your browser tab with WebAssembly + Wine — nothing is uploaded. Click the screen to capture input; press Esc to release the mouse.

How it works & what to expect

What exactly fails

The failure chain: Wine loads FREECIV_.EXE → detects a setjmp call frame → Boxedwine asserts and kills the process immediately. This happens before the window even appears. It's not a DLL issue — we bundled all 40+ required DLLs from the official installer — it's a fundamental CPU emulation gap in the Boxedwine version we use.

setjmp/longjmp lets C programs do non-local jumps across stack frames. Emulating it correctly requires Boxedwine to track shadow stack state, which is a compile-time option. The builds we use were compiled without it.

Alternatives

Freeciv's official project now ships a native web client — go to freeciv.org to play multiplayer in plain HTML with no emulation. For strategy in the browser here, try uploading an older Freeciv 2.2-era Windows SDL build that may not use setjmp, or try Age of Empires with your own disc files.

Frequently asked questions

Is Freeciv free to play?

Yes — Freeciv is GPL-2.0 open-source, completely free. It just can't run in ExeBrowser because the Win32 build uses setjmp which our Boxedwine version doesn't support.

Is Freeciv the same as Civilization?

No — Freeciv is an independent open-source game inspired by Civilization. Same genre (found cities, research tech, build an empire), separate project unrelated to Sid Meier / Firaxis.

Why does setjmp break it?

setjmp/longjmp let C programs jump back to a saved stack frame — used for exception handling and error recovery. Emulating it requires Boxedwine to save and restore complex CPU state across calls. The Boxedwine builds we host were compiled without this support, so any EXE that calls setjmp hits an assertion and dies.

Can I play Freeciv in a browser any other way?

Yes — the Freeciv project ships a native web client at freeciv.org. Play multiplayer games in plain HTML with no emulation needed.
