Can Freeciv run in your browser? Won't run — missing DLLs (libcurl, liblzma, libMagickWand) not in Wine root

Freeciv is a free, open-source empire-building strategy game inspired by Civilization. Unfortunately the Windows SDL build can't run in ExeBrowser right now: it requires several DLLs (libcurl, liblzma-5, libMagickWand) that aren't present in the Wine 1.7.55 root we use, and the older Boxedwine 18R2 runtime is missing wineboot.exe entirely. The app fails at startup with missing-module errors.

Runs in your browser tab with WebAssembly + Wine — nothing is uploaded. Click the screen to capture input; press Esc to release the mouse.

How it works & what to expect

Why Freeciv doesn't run here

The Freeciv Windows SDL client links against a large native library stack — libcurl for update checks, liblzma and libMagickWand for image handling — that goes well beyond the GDI32/USER32/KERNEL32 set that our Wine 1.7.55 root contains. Every variant we've tried ends in err:module:import_dll fatals before the main window opens.

Apps that do work are ones that only call GDI, USER32, and KERNEL32 — think Solitaire, KeePass, Notepad, SkiFree. Freeciv's dependency chain is closer to a Linux GTK/SDL desktop app, which is simply out of scope for a WASM Wine 1.7 build.

Alternatives

If you want Freeciv in a browser, the official project now ships a native web client: freeciv.org runs multiplayer games in plain HTML. For the turn-based strategy experience here, Age of Empires (bring your own disc) sometimes runs, and DOS-era 4X games like the games in the shareware era load cleanly via DOSBox. For a working browser game right now, try DOOM, Wolfenstein 3D, or SkiFree.

Frequently asked questions

Is Freeciv free to play?

Yes — Freeciv is GPL-2.0 open-source software, completely free with no DRM. However it can't run in ExeBrowser right now because its Windows SDL client requires native DLLs (libcurl, liblzma, libMagickWand) that our Wine 1.7.55 root doesn't include.

Is Freeciv the same as Civilization?

No — Freeciv is an independent open-source game inspired by the Civilization series. It plays similarly (found cities, research tech, build an empire) and ships Civ-compatible rulesets, but it's a separate project unrelated to Sid Meier / Firaxis.

Why doesn't Freeciv work in the browser here?

The Windows SDL build links against libcurl, liblzma-5, and libMagickWand — large native libraries that aren't in our Wine 1.7.55 root. Every startup attempt ends with missing-module errors before the window opens. Apps that run cleanly here only call standard Win32 GDI/USER32/KERNEL32 APIs.

Can I play Freeciv in a browser another way?

Yes — the Freeciv project now ships a native web client. Go to freeciv.org to play multiplayer games in plain HTML without any emulation needed.
