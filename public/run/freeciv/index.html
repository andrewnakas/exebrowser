Play Freeciv online in your browser Works — SDL1 client boots and is playable

Freeciv is a free, open-source empire-building strategy game inspired by Civilization. This build uses the SDL1 client (version 2.6.7) which runs under Wine + WebAssembly. Start a single-player game: click Start new game, configure your options, then click Start to begin. The SDL renderer draws directly without needing DirectX.

Runs in your browser tab with WebAssembly + Wine — nothing is uploaded. Click the screen to capture input; press Esc to release the mouse.

Freeciv is GPL-2.0 open-source. Windows builds ship as NSIS installers — install it on Windows first, then use Load your own copy above to run the installed client.

How it works & what to expect

Why an older, lighter build runs best

Modern Freeciv releases lean on newer GTK and OpenGL features that browser-based Wine 1.7.55 handles poorly. The classic 2.2 to 2.5-era Windows builds — especially the SDL client — are far friendlier: smaller, fewer modern dependencies, and closer to the Wine era we target. That's the build we host here, so you get a version that actually boots and plays rather than one that hangs at the window-creation stage.

If you'd rather bring a specific newer version, you can — but expect the modern GTK client to be hit-or-miss, and fall back to an SDL build if it refuses to open a window.

How it works & what to expect

Turn-based games are forgiving of emulation speed — there's no real-time pressure — so once Freeciv boots, play is comfortable even at 10-40% of native speed. The single-player game runs a local server in-process, so no network connection is needed to play against the AI.

The main risk is at startup: a too-modern client may fail to create its window. The hosted SDL build sidesteps that. Map scrolling, the city view, and tech tree all behave once you're in a game.

Bringing your own build

Want a different ruleset, tileset, or version? Freeciv ships as an installer, but it reads everything from disk at runtime. The cleanest path: install it on any Windows PC (or extract a portable/zip build), then copy the whole Freeciv program directory — the client EXE plus its data\ folder of rulesets and tilesets — into one folder and upload it with the folder picker. The data folder must come along, or the game won't find its rules. Official builds live at freeciv.org/download.

Frequently asked questions

Is Freeciv free to play?

Yes. Freeciv is open-source software under the GPL-2.0 license with no DRM, no paywall, and no anti-cheat. It's completely free, and because of that we can host a ready-to-run build right here in the browser.

Is Freeciv the same as Civilization?

No — Freeciv is an independent, free game inspired by the Civilization series. It plays in a very similar style (found cities, research tech, build an empire) and even ships rulesets that mimic classic Civ, but it's a separate open-source project, not a Sid Meier / Firaxis product.

Why does the newest version of Freeciv not work in the browser?

Modern Freeciv builds use newer GTK and OpenGL features that the browser's Wine 1.7.55 engine handles poorly, so they often fail to open a window. We host an older SDL-client build that boots reliably. If you upload your own copy, prefer a 2.2 to 2.5-era SDL build.

Does Freeciv need an internet connection to play?

Not for single-player. Freeciv runs a local game server in-process, so you can play against AI opponents entirely offline inside the browser. Online multiplayer against other players over the network is not supported in this sandboxed setup.

Is it fast enough to be playable?

Yes. Browser Wine runs at roughly 10-40% of native speed, but Freeciv is turn-based with no real-time pressure, so emulation overhead barely matters once a game is underway. Startup is the only slow part.
