Play Freedoom online in your browser Works well — pair with a software-rendering port

Freedoom is a fully free, open-source replacement for the DOOM data files — same legendary gameplay, zero proprietary content. It runs with the classic DOS DOOM.EXE engine in DOSBox + WebAssembly. Hit play for instant browser-based demon slaying, no download required.

D-pad: move/turn · FIRE: shoot · USE: open doors · RUN: sprint

Runs in your browser tab with DOSBox + WebAssembly — nothing is uploaded. Click the screen to capture input; press Ctrl+F10 to release mouse.

Get Freedoom (free &amp; open source)

The official, libre release

Freedoom's game data is released under a modified BSD license — you may freely copy, modify, and redistribute it, provided the license and credit travel with it. Grab the official release from the project itself:

Download Freedoom (official site) →

The download contains freedoom1.wad (Phase 1, Ultimate-Doom-style) and freedoom2.wad (Phase 2, Doom-II-style). These are WAD data files, not an executable — you also need a DOOM engine. The simplest free pairing is a small Win32 software-rendering port such as Chocolate Doom or Crispy Doom. Put the port's .EXE and a freedoomN.wad in one folder, then use “Load your own copy” above to drop the folder in. Everything here — game and engine — is genuinely free and open source.

How it works & what to expect

ExeBrowser runs a real Win32 executable on Wine + WebAssembly. Freedoom itself isn't an executable — it's an IWAD, a single data file holding all the levels, graphics, sound, and music. So the page actually runs a DOOM engine (a small Win32 .exe) and points it at freedoom1.wad or freedoom2.wad. Pick a software-rendering port and it plays at full speed on a modern laptop, exactly like the original DOOM does here.

Choose the right engine — this is the important part

The engine choice decides whether it works. Be guided by this:

  • Chocolate Doom / Crispy Doom (recommended) — software-rendering, tiny Win32 builds. No 3D acceleration needed, so they run cleanly in the emulator. This is the reliable pairing.
  • GZDoom / modern hardware ports (avoid here) — these require OpenGL 3.3 and a real GPU pipeline that Boxedwine doesn't provide. They will fail to start or render. Save those for a native desktop.

To launch, put the port's .EXE and a Freedoom WAD in the same folder. Many ports auto-detect a WAD sitting beside them; if not, the engine reads it as the IWAD on startup. Phase 1 (freedoom1.wad) is the four-episode, Ultimate-Doom-style campaign; Phase 2 (freedoom2.wad) is a single 30-level Doom-II-style campaign.

Step by step

  1. Click ▶ Play Freedoom above to boot Wine on this page.
  2. Download Freedoom from the box below and a small software-rendering port (Chocolate Doom or Crispy Doom).
  3. Unzip both into a single folder so the .EXE and a freedoomN.wad sit side by side.
  4. Use Load your own copy → a folder and select that folder, then click Run in Wine.
  5. Click the canvas to capture input. Arrows move, Ctrl fires, Space opens doors, 1–7 switch weapons, Esc releases the mouse.

What it is — and what it isn't

Freedoom is not id Software's DOOM and isn't trying to be a copy of its art — it's an original, drop-in replacement for the commercial WADs, made from scratch by volunteers so the open-source engines have something completely free to play. That makes it perfect for a browser: no shareware caveats, no licensing tightrope, no "buy the full version" upsell. If you specifically want the original 1993 DOOM levels, see our DOOM shareware page instead. If you just want a free, no-strings DOOM-style shooter to blast through, Freedoom is the cleanest option there is.

Frequently asked questions

Is Freedoom really free and legal to play online?

Yes — completely. Freedoom is released under a modified BSD license that explicitly allows anyone to copy, modify, and redistribute it freely. It contains no id Software assets, so there are no shareware or copyright caveats. Both the game data and the open-source engines you pair it with are free software.

Is Freedoom the same as DOOM?

No. Freedoom is an original, from-scratch game that runs on the classic DOOM engine and is compatible with DOOM-format levels and mods. It's a free replacement for id's commercial DOOM WADs — similar feel, all-original content. If you want the actual 1993 DOOM levels, see our DOOM shareware page.

Why do I need a separate engine to play Freedoom?

Freedoom ships as WAD data files (freedoom1.wad and freedoom2.wad), not a program. A DOOM engine reads that data and runs the game. In the browser, use a lightweight software-rendering port like Chocolate Doom or Crispy Doom — both are small, free Win32 builds that run well on Wine + WebAssembly.

Can I use GZDoom for Freedoom in the browser?

Not here. GZDoom and other modern hardware-accelerated ports require OpenGL 3.3, which the WebAssembly emulator doesn't provide, so they won't render. Stick to software-rendering ports (Chocolate Doom, Crispy Doom) in the browser, and save GZDoom for a native desktop install.

What's the difference between Freedoom Phase 1 and Phase 2?

Phase 1 (freedoom1.wad) is built like The Ultimate Doom — four episodes of eight levels each — and works with mods made for the original DOOM. Phase 2 (freedoom2.wad) is a single 30-level campaign in the style of DOOM II and works with DOOM II / Final Doom mods.
