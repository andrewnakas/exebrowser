Play Raptor: Call of the Shadows in your browser Tricky — free version is DOS, Windows port needs DirectX

Raptor: Call of the Shadows is Apogee's gloriously over-the-top 1994 vertical shoot-'em-up — buy bigger guns, blow up everything, repeat. It's a fantastic game, but it's an awkward fit for ExeBrowser, and we'd rather tell you straight than waste your time. The short version: the famous free shareware episode is a DOS game, and ExeBrowser runs Windows programs on Wine + WebAssembly, not DOS. The only Win32 version is a commercial DirectX 5 port that this old Wine engine handles poorly. So Raptor mostly belongs on a DOS emulator, not here — but read on for exactly what happens and the right way to play it.

Runs in your browser tab with WebAssembly + Wine — nothing is uploaded. Click the screen to capture input; press Esc to release the mouse.

About the versions and the licensing. The classic Episode 1 "Bravo Sector" was distributed by Apogee as free shareware and may still be shared freely — but it is an MS-DOS program (it uses the DOS4GW extender), not a Windows .exe. ExeBrowser runs Win32 binaries on Wine, so the DOS shareware will not boot here directly. The only native-Windows build is the 1999 Mountain King re-release, which is commercial (sold for around $15) and is built on DirectX 5 — exactly the kind of thing Boxedwine's old Wine struggles with. Please don't look for "cracked" or "abandonware" copies of the paid versions. If you want the full game, buy it from a legitimate source such as Steam (2015 Edition) or GOG.

How it works & what to expect

Why Raptor is a poor fit for ExeBrowser

ExeBrowser is a Windows runtime: it runs real Win32 .exe files inside Wine 1.7.55 on an emulated 32-bit x86 CPU. Raptor's problem is that its two relevant builds both fall outside that sweet spot:

  • The 1994 original (and the free shareware) is pure MS-DOS. It boots through the DOS4GW protected-mode extender and talks straight to VGA Mode-X hardware. That's a DOS program, not a Windows one — Wine has no DOS layer here, so it won't start. This is a job for a DOS emulator like DOSBox, not for Wine.
  • The 1999 Windows re-release is Win32, but built on DirectX 5. Boxedwine's bundled Wine is from the 1.7.55 era and its DirectDraw/Direct3D support is weak and incomplete, so DirectX games frequently show a black screen, garbled video, or simply refuse to start. On top of that, this version is commercial — we can't host it, and you'd need your own legally-bought copy.

That's why our verdict is partial at best. There's no free, Win32, Wine-friendly Raptor binary that lines up with all three constraints at once.

The right way to actually play Raptor

Because the freely-shareable version is DOS, the best experience is a DOS emulator, not Wine:

  1. Play the shareware in a DOS-in-browser player. The Internet Archive and other preservation sites host the freely-redistributable Episode 1 running in an in-browser DOSBox/EM-DOSBOX emulator — that's purpose-built for DOS4GW games like Raptor and runs it cleanly.
  2. Buy the full game for modern systems. The Raptor 2015 Edition on Steam and the DRM-free editions on GOG bundle a configured emulator and run all three episodes (Bravo Sector, Tango Sector, Outer Regions) on Windows, often with cloud saves and achievements.

If you specifically want a Win32 shoot-'em-up that genuinely runs well in ExeBrowser, the shareware Apogee/Epic shmups built for Windows are a far better bet — see Tyrian and Jazz Jackrabbit below.

If you want to try it here anyway

You can experiment — ExeBrowser lets you load your own folder — but go in with eyes open:

  • A DOS RAPTOR.EXE will not boot under Wine; expect it to bail out immediately or hang. This is expected, not a bug in your copy.
  • If you own the 1999 Windows port, you can load it via “Load your own copy” and try the default Win32 variant, then the r18 variant if it crashes. Be prepared for DirectX-related black screens or missing sound. We can't promise it'll get past the title screen.
  • There is also a fan-made open-source source port that re-implements the engine — but it still needs the original game's data files, so it doesn't change the licensing picture, and it isn't a drop-in Win32 build for this runtime.

None of this involves cracks or pirated copies — bring your own legally-owned files or stick to the free shareware on a proper DOS player.

Frequently asked questions

Can I play Raptor: Call of the Shadows free online with no download?

Yes — but not here. The free shareware (Episode 1, "Bravo Sector") is a DOS game, so the no-download way to play it is an in-browser DOSBox player like the one on the Internet Archive. ExeBrowser runs Windows programs on Wine, not DOS games, so the shareware won't boot in this engine.

Why won't Raptor run in ExeBrowser?

The freely-redistributable version is a DOS program (it uses the DOS4GW extender and VGA Mode-X), and Wine here has no DOS layer to run it. The only Windows-native build is the commercial 1999 re-release, which uses DirectX 5 — and Boxedwine's old Wine handles DirectX poorly, so it usually shows a black screen or fails to start.

Is the Raptor shareware legal to share and play?

Yes. Apogee released Episode 1 "Bravo Sector" as free shareware that may be copied and shared at no charge. The other two episodes (Tango Sector and Outer Regions) and the modern re-releases are commercial — buy those from Steam or GOG. Don't use cracked or "abandonware" copies of the paid content.

What's the best way to play the full Raptor on a modern PC?

Buy the Raptor: Call of the Shadows 2015 Edition on Steam or a DRM-free edition on GOG. Both bundle a pre-configured emulator and run all three episodes on modern Windows, often with cloud saves and achievements — no fiddling required.

Is there a Windows shoot-'em-up that actually runs well in the browser here?

Yes. Shareware shmups that were built for Windows (rather than DOS) tend to run far better in ExeBrowser. Tyrian and Jazz Jackrabbit are great places to start — see the related links below.
