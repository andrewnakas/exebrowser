Play JezzBall online in your browser Runs in the 16-bit (win3x) runtime — bring your own copy

JezzBall — the maddening 1992 puzzle game from the Microsoft Entertainment Pack where you wall off bouncing balls into smaller and smaller boxes — is exactly the kind of light, mouse-driven 16-bit title that ExeBrowser's Wine + WebAssembly runtime handles well. There's no DirectX, no 3D, no networking; just a single window, a mouse pointer, and balls to trap. The one real catch: JezzBall is commercial Microsoft software and we can't give you the files, so you'll need to load a copy you already legally own. Here's how to run it, which Wine variant to pick, and a few free clones if you'd rather skip the licensing hassle.

Runs in your browser tab with WebAssembly + Wine — nothing is uploaded. Click the screen to capture input; press Esc to release the mouse.

JezzBall is commercial Microsoft software and we don't host it. How it works & what to expect

JezzBall is a 16-bit Windows 3.x program from 1992. That matters here: the everyday default Win32 Wine variant is built for 32-bit apps and usually won't load a 16-bit executable. ExeBrowser ships a dedicated win3x variant for exactly this era of software, and that's the one this page is configured to use. When you load your own JEZZBALL.EXE (or the WEP launcher it came with), pick the win3x runtime.

Mechanically the game is trivial for the emulator: a small playfield, a couple of bouncing balls, and walls that grow when you click. No GPU, no sound-card gymnastics required. If your legally-owned copy starts at all, it generally plays at full, responsive speed — this is about as gentle a workload as Wine-in-the-browser ever sees.

Honest expectation: the blocker is almost never performance — it's getting a legitimate copy in the first place, since Microsoft no longer sells the Entertainment Pack. That's why we've set this page's verdict to partial: the technology works, but you have to bring your own files.

The gameplay, in 20 seconds

You're given a rectangular room with one or more balls bouncing around it. Click to start building a wall — it grows in both directions from your pointer (vertical or horizontal, toggled with a right-click or the mouse button). When a wall reaches the edges on both ends, it seals off that section; if the sealed-off part contains no ball, it fills in and shrinks the open area.

  • Clear 75% or more of the playfield to finish a level.
  • If a ball hits a wall while it's still growing, you lose a life.
  • Each new level adds another ball and ramps up the speed.

It's deceptively simple and genuinely tense — squeezing the last ball into a sliver of the board is the whole appeal.

Loading your own copy

  1. Click ▶ Play JezzBall above to boot the win3x Wine runtime in this tab.
  2. Use Load your own copy → a folder and select the directory from your legally-owned Entertainment Pack that contains JEZZBALL.EXE.
  3. Make sure the runtime is set to win3x (this page defaults to it) — the 16-bit binary won't start under the default Win32 variant.
  4. If the bundled launcher (WEP.EXE / a pack menu) doesn't cooperate, point the runtime straight at JEZZBALL.EXE instead.
  5. Once the window appears, play with the mouse — click to lay walls, the right button flips wall orientation.

If win3x crashes or hangs on launch, try the r18 variant — it uses an older Boxedwine engine that sometimes handles stubborn 16-bit programs differently.

Frequently asked questions

Can I play JezzBall online for free with no download?

You can play it in your browser with no install, but JezzBall itself is commercial Microsoft software that we don't host. You'd need to load a copy you legally own. If you want a truly free option, an open-source clone like KBounce or IceBreaker gives you the same gameplay with no licensing concerns.

Why won't JezzBall run on the default Wine variant?

JezzBall is a 16-bit Windows 3.x program from 1992. The default ExeBrowser runtime targets 32-bit Win32 apps and generally can't load 16-bit executables. Use the win3x variant (this page is set to it by default), which is built specifically for early-1990s Windows software.

Is JezzBall legal to download from abandonware sites?

It's a legal grey area we steer clear of. Microsoft has never released JezzBall as freeware and no longer sells the Entertainment Pack, so abandonware copies aren't authorized redistribution. ExeBrowser won't link to them. Load only a copy you legally own, or play a free open-source clone instead.

Will JezzBall run at full speed in the browser?

Almost certainly, if your owned copy starts at all. It's a tiny 2D game with no 3D, DirectX, or sound-card demands, so it's one of the lightest possible loads for the emulator. The hard part is obtaining a legitimate copy, not performance.

What are the controls for JezzBall?

It's mouse-driven. Click to start growing a wall from your pointer (it extends in both directions); the right mouse button toggles between vertical and horizontal walls. Don't let a bouncing ball touch a wall while it's still building, or you lose a life.
