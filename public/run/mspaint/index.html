Run classic MS Paint in your browser Classic builds work — bring your own copy

The classic Microsoft Paint (MSPAINT.EXE) that shipped with Windows 95 through XP is a tiny, single-purpose Win32 program — exactly the kind of thing browser-based Wine handles well. If you want that pixel-perfect, no-frills painting experience without installing anything, the old Paint runs nicely in ExeBrowser. Because Paint is Microsoft's property we can't host it for you, so you bring your own copy — or skip straight to the free, open-source paint tools below, which run just as well and are fully legal to share.

Runs in your browser tab with WebAssembly + Wine — nothing is uploaded. Click the screen to capture input; press Esc to release the mouse.

A licensing note

MSPAINT.EXE is part of Windows and is Microsoft's copyrighted property — we can't host it, and you should only run a copy you already own (for example, one copied from a Windows installation you have a license for). The open-source alternatives below carry no such restriction and are the easier, fully-legal route to drawing in your browser.

How it works & what to expect

The classic Paint is one of the lightest Win32 apps you can throw at browser-based Wine. The XP-era MSPAINT.EXE is essentially self-contained — drop it in and it usually launches straight away (add the MFC DLLs from the same Windows folder if it prompts for them). Later builds save to .bmp, .gif, .jpg and .png; very early Win95 builds are .bmp-only.

Better: free, open-source paint programs

You don't need a copy of Windows to paint in your browser. Several classic open-source paint tools are freely distributable and run cleanly under browser-based Wine — and unlike MSPAINT.EXE, there's no licensing question to worry about:

  • mtPaint — a tiny, fast, GPL pixel-art and image editor; a near-perfect fit for the Wine era we target and arguably nicer than classic Paint. mtpaint.sourceforge.net
  • GrafX2 — a classic 256-color pixel-art editor, very lightweight and great for sprite work. grafx2 project
  • Pinta — a Paint.NET-style editor. Note it needs .NET/Mono, so it's hit-or-miss on old Wine; treat it as experimental.

Download the program folder, load it with the folder picker, draw, save, then use Download files written by this app to get your image out as a zip.

Step by step

  1. Keep Win32 (default) selected and click Boot Wine right on this page.
  2. Use pick a folder and load mtPaint (or your own classic MSPAINT.EXE) with its files.
  3. Select the EXE entry and click Run in Wine.
  4. Draw, save your image, then click Download files written by this app before reloading to keep your work.

Frequently asked questions

Can I run MS Paint online for free with no download?

You can run the classic Win32 Paint in your browser via Wine + WebAssembly — nothing to install on our end. But MSPAINT.EXE is part of Windows and isn't ours to give out, so you supply a copy you already own. For a zero-strings option, use the free open-source mtPaint or GrafX2 instead — both run cleanly here.

Why don't you host MS Paint for download?

Microsoft Paint is copyrighted software bundled with Windows, and its license doesn't permit us to redistribute it. We only host software that's genuinely free to share. Bring your own copy of MSPAINT.EXE, or use the open-source paint tools we recommend.

Which version of Paint works best?

The Windows XP-era MSPAINT.EXE is the sweet spot — it's a self-contained Win32 program and launches with little fuss on the default variant. Very old Win95 builds work too but only save .bmp. The modern Windows 10/11 Paint is a different, heavier app and is not a good fit for browser Wine.

What's a good free alternative to MS Paint that runs here?

mtPaint is the standout — a tiny GPL image and pixel-art editor that runs cleanly under browser-based Wine and feels very close to classic Paint. GrafX2 is excellent for 256-color sprite work. Both are free to download and redistribute.

How do I save the image I draw?

Save inside the app as usual, then click Download files written by this app to pull your image out as a zip before you close or reload the tab. Anything left only inside the emulated disk is lost on reload.
